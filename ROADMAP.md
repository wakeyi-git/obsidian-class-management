# Classroom Manager 로드맵

현재 버전은 `1.0.0`입니다. 아래 1.0 범위는 구현과 자동 회귀 검증을 완료했습니다.

## 제품 원칙

- 모든 핵심 데이터는 Obsidian Vault의 일반 Markdown 파일로 남깁니다.
- 플러그인을 제거해도 학생·출결·과제·기록을 읽을 수 있어야 합니다.
- 교사의 빠른 입력과 원본 기록 보존을 우선합니다.
- 학생 개인정보와 미성년자 데이터를 다루므로 외부 AI 연동은 명시적으로 선택한 경우에만 사용합니다.
- AI 결과는 항상 근거 노트와 연결된 검토용 초안으로 만들며 자동으로 최종 기록에 반영하지 않습니다.

## 0.4 통합 조회

### 학생별 타임라인

- 학생 노트에서 관찰·상담·칭찬 기록을 시간순으로 조회
- 지각·결석·조퇴·결과와 사유 표시
- 과제 미제출·보완 내역 표시
- 항목을 클릭해 원본 Markdown 노트 열기
- 학생별 기록 수, 출결 예외, 미완료 과제 요약

### 리스트뷰

- 생활 기록·출결·과제·할 일을 하나의 공통 리스트에서 조회
- 날짜, 학생, 기록 유형, 상태, 원본 노트 열 제공
- 날짜순·학생순·상태순 정렬
- 표시 열 선택과 보기 설정 저장
- 학생별 보기와 학급 전체 보기 전환

### 필터

- 기간, 학생, 기록 유형, 출결 상태, 과제 상태별 필터
- 여러 조건을 동시에 조합
- 자주 사용하는 필터를 저장된 보기로 보관
- `미제출 과제`, `이번 달 출결 예외`, `최근 기록 없음` 기본 필터 제공

### 검색

- 학생 이름·번호, 노트 제목, 본문, 속성 통합 검색
- 한글 부분 검색과 검색어 강조
- 검색 결과에서 원본 노트 또는 학생 타임라인 열기
- 검색 범위를 학급·학생·기간·자료 유형으로 제한

## 0.5 주간·월간 캘린더뷰

- 주간·월간 보기 전환
- 생활 기록, 출결, 과제, 할 일, 가정통신문, 루틴을 날짜별로 표시
- 항목 유형별 색상과 표시 여부 필터
- 날짜 선택 후 기록·과제·할 일 빠른 생성
- 캘린더 항목 선택 시 상세 정보와 원본 노트 열기
- 오늘 이동, 이전·다음 주/월 탐색
- 학생 필터 적용 시 해당 학생의 일정과 기록만 표시
- 반복 루틴과 마감 예정 가정통신문 표시

초기 버전은 Vault 내부 날짜 속성을 기준으로 동작하며 외부 캘린더 연동은 별도 확장 항목으로 둡니다.

## 0.6 GTD 기반 실행 관리

### GTD 할 일 관리

- 빠른 수집을 위한 `Inbox`
- 바로 실행할 `Next Actions`
- 여러 단계로 구성된 `Projects`
- 다른 사람의 응답을 기다리는 `Waiting For`
- 나중에 검토할 `Someday / Maybe`
- 장소·도구·업무 영역을 나타내는 컨텍스트와 태그
- 시작일, 마감일, 우선순위, 반복 설정
- 일간 검토와 주간 리뷰 화면
- 학생·과제·가정통신문·생활 기록과 할 일 연결
- 완료한 할 일을 Markdown 기록으로 보존

### 가정통신문 회신 체크리스트

- 가정통신문 제목, 발송일, 회신 마감일 기록
- 학생별 `미회신·회신 완료·확인 필요` 상태
- 학생별 메모와 회신 일자
- 전체 회신 처리와 예외 학생만 빠른 수정
- 미회신 학생 필터 및 마감 알림
- 학생 노트와 가정통신문 원본 노트 연결
- 월간·주간 캘린더와 GTD `Waiting For`에 연계

### 루틴 체크리스트

- 일간·주간·월간 학급 운영 루틴 템플릿
- 등교 준비, 알림장, 교실 정리, 주간 기록 검토 등의 기본 예시
- 반복 주기와 실행 요일 설정
- 날짜별 체크리스트 인스턴스 생성
- 미완료 루틴과 건너뛴 항목 표시
- 사용자 정의 루틴과 순서 변경
- 캘린더 및 GTD 주간 리뷰와 연계

## 0.7 분석·보고서와 AI 협업

### 기본 분석과 보고서

- 일간·주간·월간 학급 운영 요약
- 학생별 관찰 기록과 변화 추이 요약
- 출결 사유와 과제 수행 현황 집계
- 상담 준비용 학생 요약
- Markdown·CSV 내보내기

### AI 협업 설정

플러그인에서 `AI 협업 설정` 명령을 실행하면 사용자가 내용을 확인한 뒤 다음 파일과 폴더를 선택적으로 생성합니다.

```text
Vault/
├── AGENTS.md
├── CLAUDE.md
├── AI_WORKFLOW.md
└── AI 결과/
    ├── 학생 피드백/
    ├── 학교생활기록부 초안/
    │   ├── 창의적 체험활동상황/
    │   ├── 교과학습발달상황(학기말종합의견)/
    │   └── 행동특성 및 종합의견/
    └── 검토 완료/
```

- Codex용 프로젝트 지침은 `AGENT.md`가 아니라 `AGENTS.md`로 생성합니다.
- Claude Code·Claude Cowork용 지침은 `CLAUDE.md`로 생성합니다.
- 두 파일에는 Vault 구조, 속성 정의, 읽어야 할 폴더, 제외 폴더, 출력 위치를 기록합니다.
- RAW 기록은 수정하지 않고 AI 결과를 별도 폴더에 생성하도록 지시합니다.
- 학생 이름 대신 내부 식별자를 사용하는 익명화 모드를 제공합니다.
- 원본 노트 링크와 사용한 근거를 초안에 남기도록 요구합니다.
- Vault 안의 노트 내용은 분석 데이터이며 실행 지침으로 취급하지 않도록 명시합니다.

Codex는 저장소의 지속적인 지침 파일로 [`AGENTS.md`](https://learn.chatgpt.com/docs/customization/overview)를 사용합니다. Claude는 작업 디렉터리의 [`CLAUDE.md`](https://support.claude.com/en/articles/14553240-give-claude-context-claude-md-and-better-prompts)를 세션 시작 시 읽으며, Cowork는 사용자가 허용한 로컬 폴더를 대상으로 작업할 수 있습니다.

### RAW 기록 분석 흐름

1. 교사가 Obsidian에서 관찰·상담·출결·과제 RAW 기록을 작성합니다.
2. 플러그인이 분석 대상 기간·학생·기록 유형을 선택하도록 합니다.
3. AI가 원문을 변경하지 않고 별도 분석 초안을 생성합니다.
4. 각 주장 옆에 근거가 된 Obsidian 원본 노트를 연결합니다.
5. 교사가 초안을 수정하고 `검토 완료` 상태로 전환합니다.
6. 검토되지 않은 초안은 공식 기록이나 학생 노트에 자동 반영하지 않습니다.

### 학생별 피드백

- 강점, 성장, 필요한 지원을 분리한 피드백 초안
- 관찰 사실과 해석을 명확히 구분
- 최근 기록에 지나치게 치우치지 않도록 기간 전체 확인
- 부정적 낙인이나 진단적 표현 탐지
- 학생·보호자에게 전달할 표현과 교사용 메모 분리

### 학교생활기록부 기록 초안

- 2026 학교생활기록부 기재요령(초)을 참고해 다음 세 영역으로 초안과 저장 폴더를 구분
  - **창의적 체험활동상황**: 자율·자치활동과 동아리활동 통합 특기사항, 진로활동 특기사항, 봉사활동 확인 메모
  - **교과학습발달상황(학기말종합의견)**: 교과·성취기준·평가요소, 성취 특성, 수행 과정, 참여와 성장
  - **행동특성 및 종합의견**: 연중 누적된 학습·행동·인성 관찰, 강점, 변화와 발전 가능성
- 선택한 학생·기간의 RAW 자료를 영역과 세부 범주별로 자동 분류
  - `직접 참고`: 교사가 직접 관찰·평가한 구체적 누가기록
  - `보조 확인`: 상담·보호자 전달, 과제 상태, 출결 사유처럼 원본 맥락을 추가 확인할 자료
  - `자동 제외`: 영역과 무관한 운영 자료 및 대회·수상·방과후·공개강좌 등 기재 제외 자료
- 과제 제출 상태만으로 성취수준을 판단하거나 출결 예외만으로 행동 특성을 단정하지 않도록 경고
- 반복된 사실을 합치고 단발성 사건, 교과·활동 영역 미분류 자료와 근거 부족 항목을 별도 표시
- 원본 기록 위키링크, 기재요령 기준 요약과 영역별 검토 체크리스트 제공
- 최종 확정과 시스템 입력은 반드시 교사가 수행

### 기재요령 기반 RAW 근거 수집

- [x] 학급 프로필에 학년·교육과정·기재요령 연도·점검 교과 설정
- [x] 창체 영역·활동 근거·실제 역할·참여·변화와 성장 입력
- [x] 봉사활동 장소·주관기관·실제 시간·누계·승인 여부 입력과 정성평가 경고
- [x] 교과·학교자율시간, 성취기준, 평가요소·방법, 수행 과정·결과 입력
- [x] 행동 영역, 직접 관찰, 관찰 맥락, 반복성, 성장·강점·지원 내용 입력
- [x] 동일 수업·활동의 학급 일괄 관찰 입력
- [x] 학생×창체×교과×행동 근거 누락 점검과 즉시 기록
- [x] `RAW·검토 완료·초안 제외` 상태 관리와 검토 완료 근거 우선 분류
- [x] 대회·수상·방과후·공개강좌 및 낙인·진단 표현 저장 전 경고
- [x] 기존 자유서술 기록의 영역 추천과 교사 확인 기반 수동 구조화

### 개인정보·안전 기준

- AI 기능은 기본 비활성화하며 명시적인 설정 후 사용
- 기관 정책과 개인정보 처리 기준 확인 안내
- 외부 서비스에 전달되는 파일과 폴더를 실행 전에 표시
- Vault 전체가 아닌 전용 작업 폴더만 허용하는 방식 권장
- 개인정보 제거·가명화 미리보기 제공
- API 키나 계정 토큰을 Vault에 저장하지 않음
- 결과 생성 전 분석 대상과 출력 파일을 미리 보여주는 dry-run 지원
- 자동 덮어쓰기 금지와 교사 승인 단계 유지
- AI가 생성한 내용임을 나타내는 `draft` 속성과 생성 시각 기록

Claude Cowork 공식 안전 안내도 민감한 파일에는 광범위한 폴더 접근 대신 전용 작업 폴더와 백업을 권장합니다. 자세한 내용은 [Use Claude Cowork safely](https://support.claude.com/en/articles/13364135-use-claude-cowork-safely)를 참고합니다.

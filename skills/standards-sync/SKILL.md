# 성취기준 노트·링크 동기화 (모드 B — 교사의 명시적 지시 필요)

볼트 `AGENTS.md` 모드 B·3계명 준수.

## 절차

1. **코드 유니버스 수집**: 진도표·설계 노트에서 `(?<!\[)\[(4[국수사과도음미체영]\d{2}-\d{2})\](?!\])` — 이미 링크된 `[[코드]]`는 lookbehind/lookahead 가드로 제외.
2. **전문 추출** (`VAULT/지도서/*.pdf` + 평가계획·기준안 PDF를 판본으로):
   - 각 PDF에서 `\[코드\]\s*([^\[]{10,170}?다\.)` 후보 수집(공백 정규화, 제로폭 문자 제거)
   - **무공백 골격으로 그룹 → 최빈 골격 선택**(동률이면 짧은 것) — 라틴/숫자/특수기호 포함·20자 미만·150자 초과 골격은 제외
   - **공백 복원 = 판본 교집합**: 같은 골격의 서로 다른 줄바꿈 판본들에서 모든 판본에 공통인 공백만 진짜로 인정
   - 그래도 오염이면(열 섞임·해설 오선택) 지도서 해당 쪽을 직접 읽어 수동 확정 — **절대 추정으로 짓지 않는다**
3. **노트 생성** `VAULT/학급운영/교육과정/성취기준/{코드}.md` (기존 건너뜀):
   frontmatter(`class-management: achievement-standard`, standardCode, subject, gradeBand, statement, tags) + 본문(전문 인용, `## 단원`(그 코드를 쓰는 설계 노트 링크), `## 평가(과제)`, `## 진도표`).
4. **링크화**: 진도표·설계·과제 노트에서 `[코드]`→`[[코드]]` 치환(가드 정규식 필수 — 이중 대괄호 방지).
5. **검증·보고**: 코드 수·전문 누락 0 확인, 진도표 1개 표본 왕복 파싱 검증, 치환 파일 수, 의심 전문 목록(라틴 포함·25자 미만) 출력해 교사 검수 요청.

## 함정

- 전문의 "…설정하였다" 문장은 해설이지 성취기준이 아니다 — 제외.
- 표 셀 안에서 치환해도 플러그인 파싱은 안전하지만(파이프 없음), 항상 왕복 검증으로 확인한다.

# 평가계획 → 과제 노트 (모드 B — 교사의 명시적 지시 필요)

볼트 `AGENTS.md` 모드 B·3계명 준수. 학생 확인표의 상태 값은 생성 시 "미제출"로 두고 이후 절대 수정하지 않는다.

## 절차

1. **평가계획 파싱** (`pdftotext -layout` 또는 사용자가 붙여넣은 표): 항목별로 {과목, 시기(N월 N주), 단원명, 평가 영역, 평가 요소, 성취기준 코드, 평가 방법} 추출. 과목별 총 횟수 각주와 대조해 누락 검증.
2. **날짜 산정**: 창 = N월의 (주-1)*7+1 ~ 주*7일. 해당 단원의 진도표 배정일과 교집합의 마지막 날 → 없으면 최근접 배정일. 예외 판단 기준: **실험·실기는 해당 차시 키워드 우선**(예: '소음' 차시), **단원 종합 서술 평가는 계획 시기 우선**(단원 마지막 배정일). 판단 근거를 노트 '시기' 줄에 남긴다.
3. **노트 생성** — 플러그인 `saveAssignment` 형식과 동일하게:
   - frontmatter: `class-management: assignment`, assignmentTitle(`{과목} 수행평가 - {요소}`), date, (단원 연계 시) curriculumUnitId/Title/Path, tags
   - 본문: 제목, `- 연계 단원:`(있으면), 학생 확인표(| 번호 | 학생 | 상태 | 메모 | — 학생은 `VAULT/학급운영/학생/` frontmatter에서, 상태 "미제출", 위키링크 `\|` 이스케이프), `## 평가 정보`(시기·단원·영역·성취기준·방법·원본 딥링크)
   - 단원 연계는 평가 단원이 프로젝트에 명시된 경우만(모호하면 비움 — 모달에서 교사가 지정). 파일명은 `{date} {safe(title)}.md`, **기존 파일 건너뜀**.
4. **진도표 기입**: 코어 progress 번들(parse→행의 `assignmentLink`에 `[[학급운영/과제/...|과제]]` 추가→serialize). **왕복 검증**(재파싱해 순번·배정·고정·시수·비고 동일) 실패 시 그 파일은 쓰지 않는다.
5. **루브릭 전사** (기준안 PDF 있을 때): 구조는 보통 홀수 쪽=기준안, 짝수 쪽=평가지. 스트림 모드(pdftotext, -layout 없이)가 셀 순서를 보존한다 — `평가 기준` 다음 ◎→문단→○→문단→△. 노트의 `## 평가 정보` 아래에 `### 평가 기준` 표(◎○△), `### 문항별 채점 기준`(N점 행→배점표, ◆→불릿, '배점' 문장→인용, 문항 번호는 채점표가 끝난 지점에 배치), 평가 대상·준비물, `#page=N` 딥링크. 요소 문구 매칭 실패 시 단원 번호로 폴백. **줄바꿈 가짜 공백**("만들 어")은 같은 문장이 다른 폭으로 실린 평가계획 PDF와 교차 대조로 복원(문맥 6자 윈도).
6. **보고**: 생성 목록(날짜·단원 연계 ★ 표시), 진도표 링크 행 수, 매칭 실패·불완전 추출 목록. 노트↔기준안 짝 전수 교차 검증(단원명 대조).

## 함정

- 원문 오탈자는 고치지 않고 보존한다(공식 문서 전사).
- 과제 노트를 나중에 플러그인 모달로 저장해도 `##` 섹션은 보존된다(1.20.0+) — 그 이전 버전에선 루브릭이 소실되니 버전 확인.

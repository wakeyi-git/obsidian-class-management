# Classroom Manager

학급 명단과 관찰·상담·칭찬 기록을 Obsidian의 Markdown 파일로 관리하는 커뮤니티 플러그인입니다.

## 현재 제공하는 기능

- 학급운영 폴더와 홈 노트 초기화
- 번호와 이름으로 학생 노트 생성
- CSV 명렬표 일괄 가져오기와 사전 미리보기
- 날짜별 출결 체크 및 출결부 Markdown 생성
- 과제별 제출·미제출·보완 상태와 메모 기록
- 생활 기록·출결·과제·할 일·회신·루틴을 모아보는 통합 리스트와 복합 필터·검색
- 학생별 기록·출결·과제·회신 타임라인
- 전체 운영 자료를 표시하는 주간·월간 캘린더와 날짜별 빠른 추가
- 성취기준·학생 요구·핵심 이해·평가 준거·수업·피드백·기록 초점을 잇는 교육과정 일체화 설계
- 2022 개정 교육과정 핵심 아이디어, 개념적 렌즈, 스트랜드·일반화·안내 질문 기반 개념기반 탐구학습 설계
- 차시별 수업 실행, 과정중심 평가 증거, 피드백·교사 성찰과 계획 대비 운영 시수 관리
- 연간 학사일정(휴업일·행사·단축/연장 수업일)과 요일별 기준 교시, 수업일수·수업가능시수 자동 계산
- 학기별 기초시간표와 행사·변경이 반영된 날짜별 시간표 해석, 칸 클릭으로 수정하는 주간 시간표 보기
- 과목·창체 진도표와 Excel·한글 붙여넣기 가져오기, 고정 날짜를 지키는 수업 시간 자동 배정
- 시간표·진도표·루틴·가정통신문을 결합한 주간학습안내 자동 생성
- 교과·영역별 기준·편성·실행 시수 3단 대조와 증감 허용 범위 점검
- Inbox·Next Actions·Waiting·Someday·Done 기반 GTD 할 일 보드
- 학생별 상태·회신일·메모를 저장하는 가정통신문 회신 체크리스트
- 일간·주간·월간 템플릿과 날짜별 실행 기록을 갖춘 루틴 체크리스트
- 일·주·월 및 학생별 분석 보고서, 기간 변화 점검, Markdown·CSV 내보내기
- `AGENTS.md`·`CLAUDE.md` 기반 Codex·Claude 파일 연동과 2026 기재요령 기반 학교생활기록부 영역별 검토 초안
- 창체·교과·행동 전용 RAW 근거 입력, 학급 일괄 관찰, 학생×교과 누락 점검과 근거 검토 상태 관리
- 여러 학급·학년도·학기 프로필 전환과 다음 학기 학생 명단 복사
- 학생 전입·전출·졸업 상태 및 번호·이름 변경, 이전 학급 읽기 전용 보관
- 전체 데이터 CSV, 중복 번호·깨진 학생 링크 진단, 보관 기간별 선택 정리
- 대규모 기록용 제한 동시성 색인과 통합 목록 점진 표시
- Vault 내부 시점 백업, 기존 파일 비덮어쓰기 복구, 백업 선행 레거시 마이그레이션
- 키보드 빠른 기록·할 일 수집, 모바일 단일 열 레이아웃, 포커스·스크린리더 레이블
- 관찰, 상담, 칭찬, 생활지도, 보호자 연락 기록 작성
- 학생 수, 오늘 기록, 최근 기록을 보여주는 학급 대시보드
- 저장 폴더와 학급 이름 설정
- 데스크톱과 모바일 레이아웃 지원

모든 데이터는 Vault 안의 일반 Markdown 파일로 저장됩니다. 플러그인을 제거해도 기록은 그대로 남습니다.

## 개발 환경 실행

Node.js 18 이상이 필요합니다.

```bash
npm install
npm run dev
```

Obsidian 개발용 Vault의 `.obsidian/plugins/class-management` 경로에 이 프로젝트를 두고, 커뮤니티 플러그인 설정에서 **Classroom Manager**를 활성화하세요. 실제 업무용 Vault가 아닌 별도의 개발용 Vault에서 먼저 테스트하는 것을 권장합니다.

배포용 빌드는 다음 명령으로 생성합니다.

```bash
npm test
npm run build
```

생성되는 `main.js`와 루트의 `manifest.json`, `styles.css`가 수동 설치에 필요한 파일입니다.

개발용 Vault로 바로 복사하려면 `.deploy.json`에 `{"target": "<vault>/.obsidian/plugins/class-management"}`를 만든 뒤 `npm run deploy`를 실행하세요.

사용자 설치와 업데이트는 [설치 안내](docs/INSTALLATION.md), 문제 발생 시에는 [문제 해결](docs/TROUBLESHOOTING.md)을 참고하세요.

## 첫 사용

1. 명령 팔레트에서 **Classroom Manager: 학급 공간 초기화**를 실행합니다.
2. 대시보드에서 **학생 추가**를 눌러 명단을 만듭니다.
   - 학생 한 명은 번호와 이름을 입력해 바로 추가합니다.
   - 여러 학생은 같은 모달의 **학생 명렬표 일괄 입력**에 내용을 붙여넣고 **명렬표 일괄 추가**를 누릅니다.
   - 파일을 사용하려면 모달 하단의 **CSV 가져오기**로 내용을 입력창에 불러옵니다.
   - `번호,이름`, `학생번호,성명`, `number,name` 헤더를 인식하며 헤더가 없는 2열 CSV도 지원합니다.
   - Excel이나 Google Sheets에서 복사한 탭 구분 명렬표도 붙여넣을 수 있습니다.
   - UTF-8 및 한글 Windows CSV 인코딩을 읽고, 이미 존재하는 번호는 건너뜁니다.
3. **빠른 기록**에서 학생과 기록 분류를 선택해 내용을 저장합니다.
4. **출결 체크**에서 날짜를 고르고 지각·결석·조퇴·결과 학생만 상태를 바꿉니다. 출석 이외의 상태에서는 사유를 함께 입력할 수 있습니다.
5. **과제 체크**에서 새 과제를 만들거나 기존 과제를 검색해 학생별 제출 상태와 메모를 저장합니다.
6. **할 일**에서 업무를 수집하고 GTD 상태·프로젝트·컨텍스트·마감일을 관리합니다.
7. **가정통신문**에서 학생별 회신 상태와 회신일을 확인합니다.
8. **루틴**에서 기본 예시를 만들거나 사용자 템플릿을 추가해 매일 체크합니다.
9. 리본의 학교 아이콘이나 **학급 대시보드 열기** 명령으로 현황을 확인합니다.
10. **교육과정 일체화**에서 새 통합 단원을 만듭니다. 성취기준과 학생 요구를 해석하고, 바라는 결과 → 수용할 평가 증거 → 학생 중심 학습 경험 → 피드백·기록 초점 순으로 설계합니다.
    - **개념기반 탐구학습 적용**을 켜면 핵심 아이디어, 개념적 렌즈, 매크로·마이크로 개념, 스트랜드별 일반화와 사실적·개념적·논쟁적 질문, 탐구 단계, 전이 맥락과 분석적 루브릭을 함께 설계합니다.
11. 단원 카드의 **차시 기록**에서 계획과 실제 수업, 과정중심 평가 증거, 학생 참여, 피드백과 다음 수업 조정을 남깁니다. 날짜가 있는 차시는 통합 목록과 캘린더에도 표시됩니다.
12. 단원 카드의 **학생 근거** 또는 **학생부 근거**에서 관찰 자료를 입력합니다. 통합 단원을 선택하면 교과·성취기준·평가요소·방법이 이어지고 원본 단원 및 차시 노트가 위키링크로 저장됩니다. 같은 수업은 **학생부 일괄**로 입력할 수 있습니다.
13. **분석·보고서**의 학교생활기록부 근거 누락 점검에서 학생별 창체·행동 및 교과별 근거를 확인하고, RAW 자료를 원본과 대조한 뒤 `검토 완료` 또는 `초안 제외`로 표시합니다.
14. AI 협업이 필요한 경우 **AI 협업 설정**에서 생성될 파일을 확인한 뒤 명시적으로 활성화합니다. **분석·보고서**에서 학생을 선택하고 **생활기록부 영역별 초안**을 누르면 창의적 체험활동상황, 교과학습발달상황(학기말종합의견), 행동특성 및 종합의견을 각각 또는 한 번에 만들 수 있습니다.
15. **학급·데이터**에서 새 학급·학기를 만들거나 전환하고, 내보내기·진단·보관 기간 검토를 실행합니다.
16. **백업·복구·마이그레이션**에서 수동 백업을 만들고 레거시 출결 표와 학생 링크를 정규화합니다.
17. **교육과정 운영**에서 학사일정·기준 시수·기초시간표 노트를 만들고 Markdown 표를 채웁니다. 수업일수·수업가능시수와 주간 시간표가 자동 계산됩니다.
18. **진도표 차시 가져오기**로 Excel·한글에서 복사한 차시를 붙여넣고, **진도 자동 배정**을 실행하면 각 차시가 시간표의 수업 시간에 순서대로 배정됩니다. 외부 강사 수업이나 행사 연계 차시처럼 자리가 정해진 차시는 주간 시간표의 칸 클릭 → **이 교시에 차시 고정**으로 고정합니다. 고정된 차시는 진도표 `고정` 열에 `📌`로 표시되고 위치는 `배정` 열에 기록되며, 노트에서 직접 `📌 2026-10-15(3)`처럼 적을 수도 있습니다.
19. **주간학습안내 생성**은 해당 주의 시간표·진도·아침활동(매일 루틴)·준비물·가정통신문 마감을 모아 안내문 노트를 만듭니다. 학사일정이 있으면 학급 캘린더에도 휴업일·행사가 표시됩니다.

**학생 빠른 기록**과 **할 일 빠른 수집** 명령에는 Obsidian 설정 → 단축키에서 원하는 키 조합을 지정할 수 있습니다.

## 데이터 구조

기본 설정에서는 다음 구조를 만듭니다.

```text
학급운영/
├── 학급 홈.md
├── 학생/
│   └── 01 김하늘.md
├── 기록/
│   └── 2026-07-21 143010 1 김하늘 관찰.md
├── 출결/
│   └── 2026-07-21 출결.md
├── 과제/
│   └── 2026-07-21 수학 익힘 32~33쪽.md
├── 할 일/
│   └── 2026-07-21 143010 보호자 상담 일정 잡기.md
├── 가정통신문/
│   └── 2026-07-21 현장체험학습 동의서.md
├── 루틴/
│   ├── 템플릿/
│   └── 실행/
│       └── 2026-07-21 루틴.md
├── 교육과정/
│   ├── 설계/
│   │   └── 수학 분수의 의미.md
│   ├── 수업일지/
│   │   └── 2026-07-21 수학 1차시 분수의 의미.md
│   ├── 학사일정/
│   │   ├── 2026 학사일정.md
│   │   └── 2026 기준 시수.md
│   ├── 시간표/
│   │   └── 2026 2학기 기초시간표.md
│   ├── 진도표/
│   │   └── 2026 2학기 수학 진도표.md
│   └── 주간학습안내/
│       └── 2026-08-17 주간학습안내.md
├── 보고서/
│   ├── 2026-07-21 학급 운영 월간 보고서.md
│   └── 2026-07-21 학급 운영 월간 보고서 자료.csv
├── 내보내기/
└── 백업/
```

AI 협업을 활성화하면 Vault 루트에 기존 파일을 덮어쓰지 않고 `AGENTS.md`, `CLAUDE.md`, `AI_WORKFLOW.md`, `AI 결과/` 구조를 준비합니다. 학교생활기록부 초안은 세 영역별 하위 폴더에 저장되며, 각 문서에서 RAW 자료를 `직접 참고`, `보조 확인`, `자동 제외`로 분류합니다. 과제 제출 상태만으로 교과 성취를 판단하거나 출결 예외만으로 행동 특성을 단정하지 않도록 검토 경고를 함께 제공합니다.

학교생활기록부 근거 기록은 일반 생활 기록과 같은 `기록/` 폴더에 Markdown으로 저장하되 `schoolRecordArea`, `schoolRecordSubarea`, `evidenceType`, `directObservation`, `achievementStandard`, `reviewStatus` 등의 구조화된 속성을 가집니다. 새 근거는 `raw`로 저장되며 교사가 원본을 확인해 `reviewed`로 바꾸기 전까지 초안에서 직접 참고 자료로 확정되지 않습니다. 기존 자유서술 기록은 자동 변환하지 않고 영역 추천 후 교사가 필드를 보완하는 방식으로 분류합니다.

통합 단원은 `교육과정/설계/`, 차시별 계획·실행 기록은 `교육과정/수업일지/`에 저장됩니다. 학생별 교과 근거에는 `curriculumUnitId`, `curriculumLessonId`와 원본 위키링크가 추가되어 성취기준-수업-평가 증거-기록의 일치 여부를 역추적할 수 있습니다. 설계 원리와 첨부 문서의 기능 대응은 [교육과정 일체화 설계 문서](docs/CURRICULUM_INTEGRATION.md)를 참고하세요.

개념기반 탐구 단원은 차시마다 `참여·관계 맺기 → 집중 → 조사 → 조직·정리 → 일반화 → 전이 → 성찰` 중 주된 단계를 연결합니다. 학생부 RAW 근거에는 교사가 계획한 일반화와 구분하여 학생이 실제로 표현한 `conceptualUnderstanding`, 관찰한 `inquiryProcess`, 새로운 맥락에 적용한 `studentTransferEvidence`를 저장할 수 있습니다.

학급 프로필마다 서로 다른 기본 폴더를 사용합니다. 이전 학급을 보관하면 플러그인에서 새 입력과 수정을 막고 조회·보고서·내보내기만 허용합니다. 보관 기간 정리는 파일을 즉시 삭제하지 않고 사용자가 고른 항목을 2단계 확인 후 Obsidian 휴지통으로 이동합니다.

유지관리 백업은 현재 학급 기본 폴더의 `백업/` 아래에 생성됩니다. 복구는 백업에만 있고 현재 Vault에는 없는 파일만 복사합니다. 마이그레이션은 항상 먼저 새 백업을 만든 뒤 실행합니다.

학생·생활 기록·출결부에는 `class-management` 속성과 검색 가능한 메타데이터가 들어갑니다.

## 개인정보 유의사항

학생 기록은 민감정보가 될 수 있습니다. 동기화·백업 위치와 공유 범위를 확인하고, 꼭 필요한 정보만 기록하며, 학교 및 지역의 개인정보 처리 지침을 따르세요.

AI 협업은 기본적으로 꺼져 있습니다. 활성화하더라도 플러그인이 외부 API를 직접 호출하거나 키를 저장하지 않습니다. 외부 도구에 폴더 접근을 허용하기 전에 dry-run 파일 목록과 제외 폴더를 확인하고, 모든 초안은 교사가 근거와 표현을 검토해야 합니다.

전체 데이터 처리 내용은 [PRIVACY.md](PRIVACY.md)를 참고하세요.

## 외부 연결과 권한 공개

- 네트워크 요청: 없음
- 계정·로그인 요구: 없음
- 결제·광고·텔레메트리: 없음
- 외부 실행 파일 또는 프로그램 설치: 없음
- Vault 접근: 사용자가 설정한 학급 폴더와 AI 협업을 선택한 경우의 루트 지침·결과 폴더를 읽고 씁니다.
- 삭제: 자동 삭제하지 않으며, 보관 기간 검토에서 사용자가 고르고 두 번 확인한 파일만 Obsidian 휴지통으로 이동합니다.
- AI: 외부 서비스를 호출하지 않습니다. 사용자가 Codex·Claude 등에 별도로 폴더 권한을 부여할 때만 해당 도구가 자료를 읽을 수 있습니다.

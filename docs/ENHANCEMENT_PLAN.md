## 2. 단계별 계획

학사력에 맞춘 일정: 지금은 여름방학 직전~방학 초. 2학기 개학(9월)에 실전 투입, 겨울방학(1~2월)에 신학년 도구 완성, 2027년 3월 신학년에 풀 세트로 시작하는 흐름.

### 0단계 — 기반 확보 (즉시, 반나절)

1. `git init` → 초기 커밋 → GitHub 공개 저장소 생성·push (예: `wakeyi-git/obsidian-class-management`)
2. CI 통과 확인 → `1.0.0` 태그 push → 릴리스 워크플로가 `main.js`·`manifest.json`·`styles.css` 첨부하는지 확인
3. `npm run deploy` 스크립트 추가: 빌드 후 테스트 볼트 `.obsidian/plugins/class-management/`로 자동 복사 (배포 불일치 방지)
4. (선택) Obsidian 커뮤니티 디렉터리 제출 — docs/RELEASE.md 절차 그대로

### 1단계 — 코드 건강화 (여름방학 중, ~8월)

기능 추가 전에 부채를 정리해 이후 바이브코딩 반복 속도를 지키는 단계.

- [x] 표 처리 헬퍼를 `utils.ts`로 추출 (attendance·assignment·notice 3중복 제거) — 2026-07-21
- [x] `safeFileSegment` 넓은 규칙(`#^[]` 포함)으로 단일화, `joinVaultPath`/`yamlString`/`localDate`/`booleanValue`/`csvCell`/`addOption` 사본 제거. 핵심: utils.ts에서 obsidian `normalizePath` 값 import를 순수 구현으로 대체해 테스트 대상 모듈이 utils를 공유할 수 있게 함 (obsidian은 `import type`만 허용되는 제약)
- [x] `RecordEntry.content`를 타입에서 제거 (읽는 곳 없음, 본문은 ActivityIndex가 직접 읽음. 생성 입력 `NewRecord.content`는 유지)
- [x] `schemaVersion` 매직 넘버를 `SETTINGS_SCHEMA_VERSION` 상수로 통일하고 운용 정책 주석 추가 (정규화는 멱등이라 항상 실행, 비호환 변경 시에만 버전 분기). `schoolLevel`은 초등 전용 표식으로 유지 결정
- [x] 테스트를 도메인별 15개 파일로 분리, `node --test` 러너 전환, 공용 픽스처 `tests/helpers.mjs` 추출 — 56개 명명 테스트
- [x] (선택) eslint 도입 — 2026-07-24 (1.46.0, typescript-eslint 타입 인지 린트를 check에 편입). prettier는 계속 보류
- 완료 기준: `npm run check` 통과 + 볼트 수동 스모크 테스트 (docs/TESTING.md 체크리스트)

### 2단계 — 실사용 검증 루프 (2학기, 9~12월)

이 플러그인의 최대 리스크는 코드가 아니라 "실제 교실에서 검증된 적 없음". 대형 기능 추가보다 실사용 피드백이 우선.

- 실제 학급 온보딩: 명렬표 CSV 가져오기, 루틴 템플릿 구성, 학급 프로필·백업 정책 설정
- 일상 시나리오 목표 시간을 정해 마찰 측정:
  - 아침 출결 체크 1분 이내
  - 쉬는 시간 관찰 기록(빠른 기록 단축키) 15초 이내
  - 금요일 주간 리뷰(루틴+보고서) 10분 이내
- 마찰·버그를 `docs/FEEDBACK.md`에 즉시 기록 → 주 단위로 patch 릴리스 (1.0.x)
- 모바일(폰·태블릿) 실사용 검증
- AI 협업 실전 가동: 생활기록부 근거 누적 → 학기말(12월) 영역별 초안 생성 → Claude Code/Cowork로 초안 작성 → 검토 완료 플로우를 실제로 1사이클 완주
- 개인정보 점검: 실데이터 투입 전 동기화·백업 범위 확인 (PRIVACY.md 절차)

### 3단계 — 교육과정 설계 고도화: 학사일정·시수·시간표·진도·주간학습 (11월 설계 → 겨울방학 구현 → 2027.2 완성)

남은 로드맵의 핵심. 신학년 준비(2~3월)에 가장 필요하므로 겨울방학 완성을 목표로 역산.

#### 참고 모델: 스쿨마스터 (docs/schoolmaster.pdf, 56쪽 전체 분석)

스쿨마스터의 담임 업무 흐름이 이 단계의 기능 모델이다.

> 시수편제(기준시수·증감 허용) → 학사일정·휴업일(요일별 기준시수 → 수업일수·수업가능시수 자동 계산) → 행사·단축/연장 수업일 → 기초시간표 → 연간시간표(자동 생성·조정) → 교과·창체 진도표(교과서 차시 데이터 자동 배정·재구성) → 주간학습안내(자동 생성·출력) → 교육과정 종합점검

채택 원칙:

- **담임 1인 범위만 구현한다.** 스쿨마스터의 교육과정담당자(학교 수준 편성)·클라우드 공유·동학년 수합은 4단계(CoVault 연계)로 보낸다. 학교가 확정한 값(기준시수, 학사일정)은 교사가 가져와 입력하는 대상이지 이 플러그인이 편성하는 대상이 아니다.
- **모든 데이터는 기존 원칙대로 Vault의 Markdown**(frontmatter `class-management: <kind>` + 표)으로 저장한다.
- **파생 문서(연간시간표·주간학습안내)는 원본(학사일정·기초시간표·진도표)에서 재생성 가능해야 한다.** 스쿨마스터의 "수정하면 각종 양식에 즉시 반영"을 저장형 복제 대신 "계산된 뷰 + 예외(오버라이드) 저장 + 스냅숏 내보내기"로 구현한다.
- 스쿨마스터가 행정 문서 생산에서 멈추는 지점(진도표↔실제 수업 기록의 단절)을 기존 통합단원·수업일지·생기부 근거와의 ID·위키링크 연결로 넘어서는 것이 이 플러그인의 차별점이다.

#### 3.1 학사일정·시수 기준 입력

- 학급 프로필별 `학사일정` 노트: 학기 시작·종료일, 방학, 휴업일·공휴일·단기방학, 재량휴업일을 표로 관리
- 행사 관리: 유형 구분(수업 적용 행사일=교시·과목 지정 / 전일 행사일 / 휴업), 단축·연장 수업일(해당 날짜의 교시 수 증감)
- `시수 기준` 노트: 교과(군)별·창체 영역별(자율·동아리·봉사·진로) 연간 기준시수, 학교자율시간 과목(최대 2개)과 성취기준. 2022 개정의 학년군 단위 편성을 고려해 올해·내년 시수를 함께 적을 수 있게 한다
- 요일별 기준 교시 수 설정 → **수업일수·수업가능시수 자동 계산**과 부족 경고 (스쿨마스터의 핵심 검증 로직)
- 기존 캘린더 뷰에 학사일정 레이어(휴업일·행사·방학 색상) 표시

#### 3.2 기초시간표 → 연간시간표

- 학기별 `기초시간표` 노트: 요일×교시 그리드에 과목(또는 창체 영역·전담·특별실) 배치
- 연간시간표는 **저장 문서가 아니라 계산 결과**로 구현: 기초시간표 + 학사일정(휴업·행사·단축/연장)을 결합해 날짜별 실제 교시를 산출하고, 교사가 바꾼 슬롯만 오버라이드로 저장
- 주 단위 시간표 뷰: 기초시간표와 다른 슬롯·행사 적용 교시·휴업일을 색으로 구분(스쿨마스터의 색상 문법 차용), 슬롯 교환·과목 변경 편집
- 학기·연간 집계: 과목별 편성시수 vs 기준시수 vs 증감(허용 범위 기본 ±20%, 설정 가능) 실시간 표시, 범위 초과·미달 경고

#### 3.3 교과·창체 진도표

- 과목·학기별 `진도표` 노트: 순서 있는 차시 행(단원 | 학습 내용 | 시수 | 성취기준 | 준비물 | 비고)
- 입력 경로 3가지: ① 차시 데이터셋 가져오기(3.6) ② Excel·한글 표 붙여넣기(기존 CSV/TSV 파서 재사용) ③ **통합단원 설계에서 생성**(단원의 차시 흐름 → 진도표 행, ID 연결 유지)
- **자동 배정**: 진도표 순서를 연간시간표의 해당 과목 슬롯에 순차 매핑 → 각 차시의 예정 날짜·교시 산출. 연차시(같은 날 연속 교시) 지원, 외부 강사 수업 등을 위한 **날짜-진도 고정** 플래그, 행사·휴업으로 밀린 진도의 자동 재배정
- 재구성 기록: 차시를 수정·이동·증감할 때 재구성 의도를 남기고 원 차시와 대비(스쿨마스터 재구성현황) — 통합단원의 설계 방법(교과 내 재구성 등)과 같은 어휘 사용
- 창체 진도표: 영역별 활동·주제·시수, 행사 차시 자동 반영
- 범교과 주제어 매핑: 진도 행에 주제어(안전·인성 등) 태그 → 주제어별 집계·조회

#### 3.4 주간학습안내 생성

- 주 선택 → 연간시간표 + 진도표를 결합해 요일×교시 그리드에 과목·단원·차시 학습 내용 자동 채움
- 부가 행: 아침활동(루틴 템플릿 연동), 준비물(진도표에서 수집), 알림 사항(가정통신문 모듈에서 해당 주 항목 선택)
- 산출물: `내보내기/`에 Markdown 스냅숏(생성 시각·기준 데이터 명시) + 인쇄용은 브라우저 인쇄 CSS로 대응. 한글(hwp) 변환은 플러그인 밖(외부 도구·AI 협업)에 둔다
- 주간학습안내에서 과목을 바꾸면 원본(오버라이드·진도표)에 반영 후 재생성하는 편집 동선 제공

#### 3.5 시수 종합 점검·자동 조절

- 3단 대조: **기준시수(시수 기준 노트) vs 편성시수(연간시간표) vs 실행시수(오늘까지 시간표 기준 운영 시수)** — 과목·창체 영역별 표와 증감 경고. (실행을 수업일지 완료 차시로 집계하는 초기 설계는 실사용 검증에서 폐기 — 모든 수업을 수업일지로 쓰지 않으므로)
- 기존 데이터 진단에 통합: 수업일수 미달, 기준 대비 편성 부족·초과, 진도표-시간표 슬롯 수 불일치, 진도 미배정 차시 점검 (스쿨마스터 교육과정종합점검 대응)
- 자동 조절 도우미(스쿨마스터 자동완성 마법사의 축소판): 편성시수가 기준에 어긋날 때 조정 후보 슬롯 제안, 진도표를 연간시수에 맞게 늘리거나 줄이는 제안 — 제안만 하고 적용은 교사가 확인(기존 AI·자동화 원칙과 동일)

#### 3.6 성취기준·차시 데이터셋 가져오기

- 2022 개정 성취기준 CSV/JSON 선택 가져오기 → 단원 설계·진도표·학생부 근거 입력에서 검색·선택
- 교과서 차시(진도) 데이터셋 가져오기 형식 정의(과목|단원|차시|학습 내용|시수) — 스쿨마스터처럼 내장 제공은 저작권·유지보수 부담이 있으므로, 가져오기 형식과 문서화를 제공하고 데이터는 교사가 준비
- 데이터셋 버전(교육과정 고시 연도) 필드와 변경 이력

#### 기존 모듈과의 연결 (일체화 차별점)

- 진도표 차시 ↔ 수업일지: 진도표에서 수업일지 생성 시 예정 날짜·교시·단원 ID가 미리 채워지고, 실행 완료 시 실행시수로 집계 → 계획-실행 대조가 자동으로 완성
- 학사일정·행사·진도 예정 차시를 통합 목록·캘린더에 표시 (기존 ActivityIndex 확장)
- AI 협업 지침(AGENTS.md·CLAUDE.md)에 학사일정·진도표·주간학습안내 경로와 스키마 반영 → 외부 에이전트가 주간 수업 준비(수업설계 초안)에 활용 가능

#### 비채택·보류 (스쿨마스터 기능 중)

- **NEIS 매크로 업로드**: 화면 자동 입력 방식은 안정성·정책 문제로 비채택. 대신 NEIS에 붙여넣기 좋은 텍스트·CSV 내보내기 제공

- **hwp 양식 삽입·인쇄양식 편집기**: Markdown + 인쇄 CSS로 대체, hwp는 외부 도구에 위임
- **지필·단원평가 점수 기록, 교과 종합의견 예시문 조합**: 점수 성적 처리는 범위 밖 유지. 종합의견은 기존 생기부 근거→AI 초안 흐름이 이미 대체(예시문 조합보다 근거 기반이 원칙에 부합)
- **수업도우미(발표자 뽑기·자리 정하기·타이머)**: 재미 요소로 후보에만 두고 우선순위 없음

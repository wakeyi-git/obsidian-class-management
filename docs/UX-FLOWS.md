# 화면 지도와 작업 흐름

> 마지막 갱신: 2026-07-22 (v1.23.0) · 원칙은 [UIUX-PRINCIPLES.md](UIUX-PRINCIPLES.md), 데이터 모델은 [PRODUCT.md](PRODUCT.md).

## 1. 레이아웃 — 3분할

```
┌ 왼쪽 패널 ─────┬ 중앙 ──────────────────────┬ 오른쪽 패널 ─────┐
│ 학급 메뉴       │ 뷰 (대시보드가 출발점)        │ 오늘 (상시)       │
│  · 뷰 진입점    │  탭으로 여러 뷰 병행           │ 학생 인스펙터     │
│  · 현재 뷰 작업 │                              │ 차시 인스펙터     │
│  · 빠른 작업    │                              │ (필요 시 생성)    │
└───────────────┴──────────────────────────────┴─────────────────┘
```

- **학급 메뉴**(왼쪽): 뷰 진입점 목록 + 활성 뷰의 컨텍스트 작업 + 전역 빠른 작업. 로드 시 자동 부착.
- **오늘 패널**(오른쪽): 아침 출발점 — 학사 상태, 오늘 시간표(+진도), 출결, 마감, 루틴, 다가오는 일정. 로드 시 자동 부착.
- **인스펙터**(오른쪽): 클릭한 대상(학생/교시)을 따라오는 상세 패널. 열려 있을 때만 조용히 갱신.

## 2. 뷰 지도 (16)

| 뷰 | 목적 | 주 진입 | 대표 액션 |
|---|---|---|---|
| 학급 대시보드 | 오늘 요약·명단 | 리본·메뉴 | 명단 클릭→학생 인스펙터 |
| 오늘 | 하루 운영 콕핏 | 자동 | 교시 좌클릭→차시 인스펙터, 우클릭→변경 메뉴 |
| 교육과정 운영 | 주간 시간표·시수 점검·설정 4종 | 메뉴 | 칸 좌클릭 보기/우클릭 편집, 진도 자동 배정, 주간안내 생성 |
| 교육과정 로드맵 | 학기 단원·평가·행사 타임라인 | 메뉴 | 막대→단원, ◆→과제, ●→행사, 학기·통합만 토글 |
| 교육과정 일체화 | 단원 목록·설계 현황 | 메뉴 | 새 단원 설계, 수업일지, 근거 기록 |
| 차시 인스펙터 | 한 교시의 모든 것 | 교시 클릭 | 이 교시 편집·수업일지 쓰기/열기·학생부 근거·진도표 |
| 학생 인스펙터 | 한 학생 30일 요약 | 명단 클릭 | 빠른 기록·근거·타임라인 |
| 학급 캘린더 | 월간 일정 | 메뉴 | 날짜별 기록 진입 |
| 통합 목록 / 타임라인 | 모든 활동 검색·학생별 서사 | 메뉴 | 필터·저장된 보기 |
| GTD 할 일 / 루틴 | 교사 업무·체크리스트 | 메뉴 | 빠른 캡처 |
| 분석·보고서 / 학급·데이터 / 백업 | 집계·설정·안전 | 메뉴 | 보고서 생성·백업 |

## 3. 핵심 워크플로

### 0. 학년·학기초 — 교육과정 운영 계획 수립 (2월 말·8월 중순, 반나절~)

> 완료 지표: 교육과정 운영 뷰의 설정 4종이 모두 ✓, 시수 점검 전 과목 '적정', 교육과정 로드맵에 전 단원 표시, 모든 수행평가가 진도표 과제 열에 링크.

1. **학급 준비** — `작업 공간 초기화` → 학급 프로필(학년도·학기·과목 목록) → `CSV 명렬표 가져오기`.
2. **계획 원장 3종** — 교육과정 운영 뷰의 체크리스트를 위에서부터: ① 학사일정(학기 시작·종료, 휴업일, 행사) ② 기준 시수 ③ 기초시간표. 입력 즉시 수업일수·수업가능시수가 계산된다.
3. **진도표** — `진도표 차시 가져오기`(엑셀·한글 붙여넣기) → `진도 자동 배정` → 시수 점검에서 부족·잉여 0 확인. 행사와 겹치는 차시는 이 단계에서 📌 고정.
4. **단원 설계** — 일반 단원은 지도서 전개(차시 흐름·성취기준)를 따라, 프로젝트는 통합 단원(개념기반: 렌즈→스트랜드→일반화→안내 질문→GRASPS)으로 설계한다. 재구성으로 프로젝트에 이관되는 차시는 일반 단원 전개에 `→ 통합 운영` 표시를 남기고, 발표회·체험학습 같은 실제 행사를 GRASPS의 무대로 연결한다. **교육과정 로드맵**으로 학기 전체의 단원 배치·겹침·공백을 검토한다.
5. **평가 계획** — 학교 평가계획의 시기·요소를 근거로 과제 노트를 만든다(날짜는 해당 단원의 배정 차시에 맞춤, 단원 연계 지정, 루브릭·평가지 첨부). 저장하면 진도표 과제 열에 자동 기입된다. 간트의 ◆ 분포로 평가 몰림을 점검한다.
6. **연계 마감** — 성취기준 노트·위키링크 확인, `일체화 Bases 보기 만들기`, `행사 노트 일괄 만들기`, 첫 주 `주간학습안내 생성`.

이후는 아래 운영 주기로 넘어간다. 학기 중 계획 변경은 이 단계로 돌아오지 않고 우클릭 편집(시간표)·단원 노트 수정으로 흡수된다.

> **실행 경로(2026-07-22)**: 전 단계 플러그인 단독 경로 완비 — 4는 `일반 단원 일괄 생성`, 5는 `평가 계획 가져오기`, 6은 `성취기준 노트 생성`·`진도표 성취기준 링크화`(R0 구현). 지도서 전문 반영·루브릭 전사가 필요하면 **LLM 협업 경로가 더 깊다**([PRODUCT.md §9-1](PRODUCT.md), 2026-2학기 실사례) — 단독 경로는 그 결과를 대체하지 않고 보완한다.

### 운영 주기 1~6

1. **아침(1분)** — 오늘 패널: 학사 상태·시간표·미체크 출결 확인 → `출결 체크`. 다가오는 행사 클릭해 준비물 확인.
2. **수업 변경(10초)** — 시간표 칸 **우클릭** → 과목 즉시 변경/수업 삭제/변경 제거. 저장 즉시 영향 과목 진도 자동 재배정 알림.
3. **수업 후 기록(30초~)** — 교시 우클릭 `수업일지 쓰기`(맥락 자동 채움, 단원 자동 제안) 또는 차시 인스펙터 `학생부 근거`(학생 검색→단원·수업일지 프리필). 다음 날 자동 RAW 확정.
4. **평가일** — 진도표·간트의 `과제` 링크 → 과제 노트(루브릭·채점 기준·평가지 딥링크) → 확인표 체크(아래 섹션은 보존됨).
5. **주간 준비(금요일)** — 교육과정 운영: 다음 주 시간표 점검, 행사 차시 📌 고정 확인 → `주간학습안내 생성`. 간트로 단원·평가 흐름 확인.
6. **학기말** — RAW 근거·수업일지 → 학생부 근거 검토(raw→검토) → 보고서·**AI 협업 내보내기**(익명화)로 생기부 초안 생성 → `AI 결과/` 초안을 교사가 검토·확정. 백업 후 종료.

## 4. 진입 규약

- 하나의 대상에는 **보기(좌클릭)**와 **편집(우클릭 메뉴)** 두 경로만 둔다. 새 기능도 이 두 경로에 편입한다.
- 노트로 존재하는 개체(단원·과제·행사·근거·성취기준)는 어느 화면에서든 **클릭하면 그 노트**가 열린다.
- 명령 팔레트는 모든 진입점의 백업 경로다(37개, `클래스` 검색).
- **모바일(터치)**: 탭=보기, **길게 누르기=편집 메뉴**(우클릭 등가) — 규약은 확정, 구현은 로드맵([UIUX-PRINCIPLES.md §7](UIUX-PRINCIPLES.md)). 구현 전까지 모바일에서는 인스펙터의 버튼 경로를 사용한다.

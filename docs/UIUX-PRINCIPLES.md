# UI/UX 원칙

> 마지막 갱신: 2026-07-22 (v1.23.0) · 새 기능·수정은 이 원칙에 맞춘다. 원칙과 코드가 어긋나면 §7 백로그에 기록하고 릴리스 단위로 해소한다.

## 1. 상호작용 규약

- **좌클릭 = 보기, 우클릭 = 편집.** 시간표·오늘 패널의 교시가 표준: 좌클릭(또는 Enter/Space)은 인스펙터, 우클릭은 컨텍스트 메뉴. 같은 클릭이 두 가지 일을 동시에 벌이지 않는다.
- **컨텍스트 메뉴 구성 순서**: 즉시 선택지(과목 목록 등, 현재값 ✓) → 상태 동작(삭제·복원, 파괴적은 `setWarning`) → 연계 동작(수업일지·진도 고정) → `편집…`(모달). 모달은 목록에 없는 입력·사유가 필요할 때의 최후 경로다.
- **모달을 여는 컨트롤 라벨**은 대상이 자명하면 `편집…`, 아니면 "동사+대상"(예: `수업일지 쓰기`). 이미 있으면 `열기`, 없으면 `쓰기/만들기`로 동사를 구분한다.
- **키보드**: 클릭 가능한 행·칸은 `role="button"`+`tabindex=0`+Enter/Space 동등 동작. `aria-label`은 "클릭: X, 우클릭: Y" 형식으로 두 경로를 안내한다.
- **원클릭 맥락 채움**: 기록 진입(수업일지·근거·과제)은 현재 맥락(날짜·교시·과목·단원·차시)을 미리 채워서 연다. 사용자가 다시 고를 수 있게 하되 기본값이 정답에 가깝게.

## 2. 용어 사전 (단일 용어 원칙)

| 표준어 | 뜻 | 금지된 유의어 |
|---|---|---|
| **수업일지** | curriculum-lesson, 차시의 기록 허브 | 수업 기록, 차시 기록, 이 교시 기록 |
| **과제** | assignment 기능·열·링크 라벨 | 과제(평가), 평가 과제, 수행평가(라벨로는 금지 — 노트 제목의 고유명으로만) |
| **단원** | curriculum-unit 총칭 (구분: 일반/통합) | "통합 단원"을 총칭으로 쓰는 것 |
| **학생부 근거** | record 중 근거 기록 (상태 수식은 괄호: "(RAW)") | RAW 근거(단독) |
| **진도표 / 차시** | 원장 노트 / 그 행 | — |
| **행사 / 휴업일** | 수업일 내 이벤트 / 비수업일 | — |
| 상태어 | 계획→실행 완료(수업일지), raw→검토(근거), 제출/미제출/보완(과제) | — |

새 문자열을 추가할 때 이 표에 없는 개념이면 표부터 갱신한다.

## 3. 알림(Notice) 문형

- 성공: `대상 + 결과`를 완결형으로. 파급이 있으면 덧붙인다 — 예: `10-15 3교시 → 체육 (2학기) · 진도 재배정`.
- 실패: `…하지 못했습니다.` + 원인 메시지 그대로. 자동 동작(auto)은 실패를 조용히 삼키되 다음 기회에 재시도되는 구조여야 한다.
- 안내: 다음 행동을 포함한다 — 예: "…`실시 차시 확정` 명령으로 일괄 생성할 수 있습니다."

## 4. 시각 언어

- **색은 테마 변수만**: 강조/통합=`--interactive-accent`, 위험/오늘=`--color-red`, 과제 마커=`--color-orange`(◆), 행사 마커=`--color-purple`(●), 보조 텍스트=`--text-muted`. 하드코딩 금지(다크·라이트 자동 대응).
- **아이콘은 lucide**, 개념당 하나 고정: 수업일지 `notebook-pen`, 단원 `book-open-check`, 통합 표식 `sparkles`, 고정 📌(텍스트), 간트 `gantt-chart`.
- **표**: `table-layout: fixed`, 첫 열(교시 등)은 내용 폭 고정, 나머지 균등. 긴 내용은 줄바꿈.
- **빈 상태**는 다음 행동을 문장으로 안내한다("…을 입력하면 나타납니다").

## 5. 데이터 안전 UX

- 플러그인이 노트를 다시 쓸 때 **사용자 섹션을 보존**한다(과제 확인표 아래 `##` 등). 새 재작성 경로를 만들면 보존 규칙부터 정한다.
- 파괴·대량 동작은 사전 안내와 사후 요약(건수)을 보이고, 대량 자동 생성은 임계(14일 규칙처럼)를 넘으면 명령 실행으로 위임한다.
- 사람이 고칠 수 있는 형식(표·frontmatter)을 깨뜨리는 최적화는 하지 않는다.

## 6. 코드 층위 규약 (요약)

- core는 옵시디언을 모른다(`import type`만, 가드 테스트 `tests/core-purity.test.mjs`).
- 볼트 IO는 repository로만. 뷰·모달은 plugin 메서드를 호출한다.
- 노트 본문·Bases 정의 등 직렬화 템플릿은 core(`entity-notes`, `curriculum` 등)가 단일 진실 — 스크립트·플러그인이 같은 함수를 쓴다.

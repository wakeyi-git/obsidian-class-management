# Classroom Manager 제품 정의서

> 마지막 갱신: 2026-07-24 (v1.48.0) · 이 문서는 구현된 코드를 근거로 작성된 사실 기술이며, 새 기능은 이 문서와 [UIUX-PRINCIPLES.md](UIUX-PRINCIPLES.md)에 맞춰 설계한다.

## 1. 한 줄 정의

초등 담임교사 1인이 **교육과정-수업-평가-기록의 일체화**를 옵시디언 볼트 안에서 계획하고, 운영하고, 증거로 남기게 하는 플러그인.

## 2. 목표

1. **일체화**: 성취기준 → 단원 설계 → 차시(진도) → 수업일지 → 평가(과제) → 학생부 근거가 클릭으로 오가는 하나의 그래프가 된다.
2. **운영 부담 제로에 수렴**: 계획(학사일정·시간표·진도표)을 한 번 세우면 배정·시수·주간안내는 계산으로 따라온다. 교사는 변경과 기록만 한다.
3. **기록의 장기 보존**: 모든 데이터는 사람이 읽고 고칠 수 있는 마크다운으로, 플러그인이 사라져도 열린다(보존 연한 5년 이상).

## 3. 비목표 (하지 않는 것)

- NEIS 대체·자동 전송, 성적 산출/통지표, hwp 생성
- 학교·학년 단위 공유와 소통 → **CoVault**의 역할 (역할 분담: 플러그인=교사 개인 운영·마크다운, CoVault=공유·DB)
- 전 교시 자동 기록 노트(빈 노트는 소음 — §6 원칙 3)

## 4. 사용자·환경

- 사용자: 담임교사 1인(현재 3학년), 학급 1~n개(프로필 전환)
- 환경: 옵시디언 데스크톱·모바일(`isDesktopOnly: false`), 데이터는 볼트 `학급운영/` 아래
- 유지보수: AI 협업 기반 솔로 개발 — 단순함과 테스트(147개)·린트가 유지 가능성의 핵심
- **공동 작업자로서의 LLM**: 개발(바이브 코딩)과 운영(계획 수립·초안 작성·데이터 추출) 모두에서 LLM은 명시적 협업자다. 협업 모델은 §9-1, 규약은 [UIUX-PRINCIPLES.md §6-1](UIUX-PRINCIPLES.md).

## 5. 계층 구조

```
packages/core/          순수 도메인 로직 (플랫폼 독립 — obsidian은 import type만, DOM·볼트 IO 토큰도 가드 테스트로 금지)
  학사일정·시간표·진도·단원·시수감사·주간안내·노트 직렬화/파싱 등 23모듈
src/
  class-repository.ts   볼트 IO 단일 창구 (폴더 규약·노트 생성/파싱·frontmatter·AI 작업 공간·백업/복구/이행)
  curriculum-flows.ts   교육과정 도메인 플로 (배정·주간안내·성취기준·기록 흐름 — 뷰는 플러그인 위임 경유)
  commands.ts           명령 팔레트 42개 등록
  *-view.ts (16)        중앙·사이드 뷰 (ItemView)
  *-modal.ts (20)       입력 모달 (+modals.ts)
  main.ts               조립: 설정·프로필·뷰 레지스트리·open 흐름 (~1,000줄)
```

의존 방향: `views/modals → main·repository → core`. 역방향 금지(core는 아무것도 모른다).

## 6. 아키텍처 3원칙

1. **노트가 진실이다** — 저장소는 마크다운 노트뿐. 표는 곧 편집 UI이므로 사람이 직접 고칠 수 있고, 플러그인은 그것을 다시 읽는다(헤더 이름 기반 열 해석으로 손편집 허용).
2. **원장은 계산한다** — 배정 날짜·시수 집계·활동 색인 같은 파생 값은 저장하지 않고 매번 계산한다. 시간표가 바뀌면 영향 과목만 재배정한다. (쓰기 증폭 방지의 근거)
3. **과거는 불변, 미래는 계획** — 날짜가 지난 기록(수업일지 등)은 `recordStatus: raw`로 확정하고 플러그인이 다시 쓰지 않는다. 노트는 사람이 적거나 링크될 때만 만든다(온디맨드).

## 8. 기능 맵 (4도메인)

1. **학급 기반**: 명단(CSV 가져오기)·출결·기록·할 일·통신문·루틴·다학급 프로필
2. **교육과정 운영**: 학사일정→수업일수·가능시수, 기초시간표+주간 시간표(우클릭 편집), 진도 자동 배정+📌 고정, 시수 점검, 주간학습안내 생성
3. **일체화**: 단원 설계(일반 43+통합 6)·수업일지 허브·과제 연계·학생부 근거 연결·성취기준 노트·Bases 보기·교육과정 로드맵
4. **분석·유지**: 대시보드·통합 목록·타임라인·보고서·백업/복구·AI 협업 내보내기

## 9. 일체화 그래프 (연계의 표준 경로)

```
[[성취기준]] ←코드 링크— 진도표 행 —단원·영역→ [[단원]] (일반/통합)
     ↑                          │ 배정(계산)              │ 연계 기록.base 임베드
     └── [[과제]] —연계 단원→ ───┘←— [[수업일지]] —단원 연계(선택)

        [[학생부 근거]] —단원·수업일지 링크→ (raw→검토→생기부 초안)
```

## 9-1. LLM 협업 모델 (정식 실행 경로)

플러그인의 모든 작업에는 두 실행 경로가 있고, 둘 다 정식이다:

| 경로 | 수행자 | 적합한 일 | 안전장치 |
|---|---|---|---|
| **단독 경로** | 교사+플러그인 UI | 일상 운영(출결·기록·변경·체크) | 플러그인 검증·Notice |
| **LLM 협업 경로** | 교사+LLM(볼트 직접 또는 내보내기) | 일괄 생성·추출·초안(단원 스캐폴드, 평가계획 파싱, 성취기준 추출, 생기부 초안) | §6-1 협업 규약 |

- **내장 AI 협업(내보내기형)**: `AI 협업 설정` → 익명화(`aiAnonymizeStudents`)·제외 폴더를 적용한 RAW 묶음 내보내기 → 외부 LLM 분석 → 결과는 `AI 결과/`(kind `ai-draft`)에 초안으로 반입. 초안은 교사 검토 전까지 공식 기록에 반영되지 않는다.
- **볼트 직접 협업(에이전트형)**: Claude Code 등 에이전트가 볼트·저장소를 직접 읽고 쓴다. 2026-2학기 준비(단원 43·과제 34·성취기준 144 일괄 생성)가 이 경로의 실사례다. 에이전트는 플러그인과 **같은 코어 직렬화 함수**를 사용하고, 왕복 검증·백업·멱등성을 지켜야 한다(규약 §6-1).
- **역할 경계**: LLM은 초안·일괄·추출을 맡고, **확정·평가·판단은 항상 교사**가 한다. 이 경계는 데이터에도 반영된다(ai-draft/raw→검토 상태).

## 10. 로드맵과 미구현 경계

> 설계상 존재하지만 아직 플러그인 기능이 아닌 것들. 새 구현은 이 절의 우선순위를 따른다. (과거 백로그는 [ENHANCEMENT_PLAN.md](ENHANCEMENT_PLAN.md))

### R0 — 계획 수립 도구화 ★2027 학년초 전 필수 (2026-07-22 구현 완료)

워크플로 0단계 중 4(단원 설계)·5(평가 계획)는 2026-2학기 준비에서 **LLM 협업 경로**(지도서 PDF 추출·일괄 생성)로 수행했다 — 이는 §9-1의 정식 경로다. R0의 목적은 이를 대체하는 것이 아니라 **LLM 없이도 가능한 단독 경로를 보완**하는 것이다(협업 경로가 가능할 땐 그쪽이 더 빠르고 깊다):

1. **일반 단원 스캐폴드** — 진도표를 읽어 과목·단원별로 단원 노트 초안(전개=차시 흐름, 기간·시수, 성취기준 코드) 일괄 생성. 지도서 전문은 수기 보완 전제. ✅ `일반 단원 일괄 생성` 모달(과목 선택·기존 건너뜀 미리보기).
2. **평가 계획 가져오기** — 진도표 가져오기와 같은 붙여넣기 UX: 학교 평가계획 표(시기·단원·요소·기준)를 붙여넣으면 과제 노트 일괄 생성+단원 연계+진도표 기입. ✅ `평가 계획 가져오기` 모달(주차 시기→배정 차시 날짜 매칭).
3. **성취기준 도구** — 성취기준 노트 스캐폴드(코드→노트)와 `[코드]`→`[[코드]]` 링크화 명령. 전문 입력은 수기 또는 데이터셋 연동(장기). ✅ `성취기준 노트 생성`·`진도표 성취기준 링크화` 명령.

### R1 — 원칙의 일관 적용

- **불변(raw) 확정의 범위 확장 검토**: 현재 수업일지만 자동 raw. 행사(경과 후)·과제(평가 완료 후)에도 같은 수명주기를 적용할지 결정하고 §6 원칙 3을 갱신.
- **대량 파생 쓰기 전 자동 스냅숏**: 진도 자동 배정처럼 수백 행을 다시 쓰는 동작 앞에 대상 노트 백업(백업 뷰 재사용). ✅ 2026-07-22 — 자동 배정·성취기준 링크화에 적용, 백업 폴더 접미사 "자동".
- **용어 가드 테스트**: UIUX 용어 사전의 금지 유의어를 소스 문자열에서 검출하는 테스트로 §2를 코드처럼 강제. ✅ 2026-07-22 `tests/terminology.test.mjs`.

### R2 — 운영 편의

- 학기 경계에서 설정 `semester` 자동 전환 제안(날짜 기반), 수동 작성 수업일지의 진도표 비고 역링크, 성취기준 노트의 플러그인 인식(검색·연결 UI), 터치(모바일) 상호작용(UIUX §1-모바일).

### R3 — 생태계 다리 (12월 재검토와 연동)

- **CoVault 내보내기**: 주간학습안내·알림장을 CoVault 형식으로 내보내는 단방향 다리(플러그인=마크다운 축, CoVault=공유 DB 축 역할 분담의 실행).
- 커뮤니티 플러그인 등록(사용자 수동 절차 문서화).
